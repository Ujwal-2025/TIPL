### Core Documentation (Read in Order)

1. **[TECH_STACK.md](./TECH_STACK.md)** - Technology Guide
   - **What it covers**: All technologies used (Next.js, React, tRPC, Prisma, etc.)
   - **Who should read**: Everyone, especially beginners
   - **Time**: 30-60 minutes
   - **Read when**: Learning the project for the first time

2. **[DEVELOPER_GUIDE.md](./DEVELOPER_GUIDE.md)** - Development Manual
   - **What it covers**: Setup, daily workflow, coding standards, common tasks
   - **Who should read**: All developers
   - **Time**: 30 minutes
   - **Read when**: Before starting development

3. **[ARCHITECTURE.md](./ARCHITECTURE.md)** - System Design
   - **What it covers**: System architecture, data flow diagrams, design patterns
   - **Who should read**: Developers who want to understand the big picture
   - **Time**: 30 minutes
   - **Read when**: Understanding how everything connects

4. **[GIT_WORKFLOW.md](./GIT_WORKFLOW.md)** - Version Control Guide
   - **What it covers**: Git commands, commit conventions, branching
   - **Who should read**: Everyone who commits code
   - **Time**: 20 minutes
   - **Read when**: Before making your first commit

5. **[QUICK_REFERENCE.md](./QUICK_REFERENCE.md)** - Cheat Sheet
   - **What it covers**: Common commands and code snippets
   - **Who should read**: Everyone (keep this open while coding!)
   - **Time**: 5 minutes to skim
   - **Read when**: Daily reference
